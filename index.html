DRY

Don't repeat yourself

What is DRY?

DRY is a software development principle that was mentioned and defined by Andy Hunt and Dave Thomas in their book The Pragmatic Programmer in 1999.

Examples of non DRY and DRY code


							const vegetables = [cucumber, carrot, potato, tomato, pumpkin];

							// Non DRY code
							console.log(vegetables[0]);
							console.log(vegetables[1]);
							console.log(vegetables[2]);
							console.log(vegetables[3]);
							console.log(vegetables[4]);

							// DRY code
							for (let i = 0; i < vegetables.length; i++) {
								console.log(vegetables[i]);
							}

Examples of non DRY and DRY code


							const vegetables = [cucumber, carrot, potato, tomato, pumpkin];
							const fruit = [apple, pomegranate, banana, peach];

							// Non DRY code
							console.log(vegetables[0]);
							console.log(vegetables[1]);
							console.log(vegetables[2]);
							console.log(vegetables[3]);
							console.log(vegetables[4]);

							console.log(fruit[0]);
							console.log(fruit[1]);
							console.log(fruit[2]);
							console.log(fruit[3]);

							// DRY code
							function printFood (array) {
								for (let i = 0; i < array.length; i++) {
									console.log(array[i]);
								}
							}

							printFood(vegetables);	
							printFood(fruit);

Examples of non DRY and DRY code


							// Non DRY code
							const a = [BMW, Opel, Ford, Tesla, Fiat];

							function some (smth) {
								for (let i = 0; i < smth.length; i++) {
									console.log(smth[i]);
								}
							}

							some(a);

							// DRY code
							const cars = [BMW, Opel, Ford, Tesla, Fiat];

							function printItems (array) {
								for (let i = 0; i < array.length; i++) {
									console.log(array[i]);
								}
							}

							printItems(cars);

Why is knowledge duplication a bad practice?

  • Changes need to be made in each place where data is duplicated;
  • Long code;
  • Code takes up more disk space;
  • It's difficult to find something in such code.

How does duplication arise?

Most of the duplications occur for the following reasons:

  • Imposed duplication;
  • Inadvertent duplication;
  • Impatient duplication;
  • Interdeveloper duplication.

What are the advantages?

Advantages:

  1. Maintainability;
  2. Readability;
  3. Reuse;
  4. Cost;
  5. Testing.
